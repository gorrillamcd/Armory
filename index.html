Armory is a web application making it easy for people to deploy a Bible College either on the internet or local network. It is written with Ruby on Rails and is currently still cooking in the oven. Don't worry, it'll be ready soon. If you'd like to contribute, check out the source code.

Why Armory?

At the Bible College where I serve, we were wanting to make our classes available online for people to take. Plus, we were already feeling the hurt of not having a cohesive solution for managing students, teachers, etc. So we started looking at different solutions, but none seemed to fit. Either they were pricey with lots of features we didn't need or they were not high-quality and easily customizable.

So, at the end of 2011, we started building our own solution, and so Armory was born...

The Vision

We hope that Armory will be used to make life easier for Bible Colleges and other ministries that have need of this type of application. It will not only be for making courses available online, but also for local administration of a physical bible college. The goal is to create a versatile application to function both as a student management system and online learning platform that looks professional, is packed with only the features you need, and that can be operated by someone without programming knowlegde

Version 1

Version 1 is currently being worked on. It will provide the basic functionality necessary for an online school system (courses, secure login system, quizzes and exams, lessons, etc) but will be lacking in some handy features like communication with the teacher within the application, forums for discussion of the lessons, the ability to customize easily the look and branding and various admin features. That's where version 2 comes in! Version 1 is also very specific to CCBC Mexico. Version 2 will be more generalized and customizable.

Contributing to the Project

Are you knowledgeable with web development? How about Ruby on Rails? Then you can help develop this project. Even if you aren't a programmer, you can help with testing or become a programmer ( it's not as hard as you might think! )

Contributing code is easy:

  • Fork the repository
  • Make the change you want to incorporate
  • Make a pull request
  • It will then be incorporated or not

Also, if you find a bug, but don't know/don't have time to make the change yourself, post an issue on the issues page.

Do you want to try out the application, but don't have a development environment setup? Check the wiki for a tutorial of how to setup the dev environment (even for windows newbies!).
